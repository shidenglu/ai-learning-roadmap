# 汇编（Assembly）

# 1. 汇编阶段概述

汇编（Assembly）是编译过程中的第三个阶段。

它的作用是：

```text
将汇编语言转换为机器指令（二进制代码）
```

即：

```text
汇编代码
      ↓
机器码
```

在整个编译流程中的位置：

```text
main.c
  ↓
预处理
  ↓
main.i
  ↓
编译
  ↓
main.s
  ↓
汇编
  ↓
main.o
  ↓
链接
  ↓
main.elf
```

---

# 2. 汇编阶段的输入与输出

## 2.1 输入文件

汇编阶段输入：

```text
main.s
```

即：

```text
汇编源文件
```

例如 ARM 汇编：

```asm
.global add

add:

    ADD r0,r0,r1

    BX lr
```

---

## 2.2 输出文件

生成：

```text
main.o
```

称为：

```text
目标文件(Object File)
```

它包含：

* 机器指令
* 数据
* 符号信息
* 重定位信息
* 调试信息

---

# 3. 汇编器（Assembler）的作用

执行汇编工作的程序叫：

```text
Assembler
```

常见：

| 工具               | 平台          |
| ---------------- | ----------- |
| GNU as           | GCC工具链      |
| arm-none-eabi-as | ARM裸机       |
| clang assembler  | LLVM        |
| MASM             | Windows x86 |

---

# 4. 汇编阶段做什么

汇编器主要完成：

```text
汇编指令解析
↓
指令编码
↓
生成机器码
↓
生成目标文件
```

---

# 5. 汇编指令转换为机器码

## 5.1 示例

ARM汇编：

```asm
MOV r0,#10
```

含义：

```text
将立即数10放入寄存器r0
```

汇编器转换：

```text
机器码

11100011 10100000 00000000 00001010
```

CPU最终执行：

```text
0xE3A0000A
```

---

# 6. CPU为什么不能直接执行汇编

人类：

```asm
MOV r0,#10
```

CPU：

```text
1011 0010 0101 ...
```

CPU只能识别：

```text
二进制机器指令
```

所以：

```text
汇编语言
↓
汇编器
↓
机器码
↓
CPU执行
```

---

# 7. 指令编码过程

以 ARM 为例：

汇编：

```asm
ADD r0,r1,r2
```

表示：

```text
r0 = r1 + r2
```

汇编器需要确定：

## 目标寄存器

```text
Rd = r0
```

---

## 第一个操作数

```text
Rn = r1
```

---

## 第二个操作数

```text
Rm = r2
```

---

然后按照 ARM 指令格式：

```
31                         0
+---------------------------+
|opcode|Rn|Rd|Rm|condition |
+---------------------------+
```

生成：

```text
32bit机器指令
```

---

# 8. 符号处理（Symbol）

汇编阶段会记录程序中的符号。

例如：

main.s：

```asm
.global add

add:

    ADD r0,r0,r1

    BX lr
```

汇编器记录：

```
Symbol Table

add
地址:
未知
```

---

原因此时：

```text
add函数最终地址还不知道
```

因为可能还有：

```text
其他.o文件
库文件
```

需要链接阶段确定。

---

# 9. 重定位信息（Relocation）

## 9.1 为什么需要重定位

例如：

main.c：

```c
extern int add(int,int);


int main()
{
    add(1,2);
}
```

编译后：

```asm
BL add
```

但是：

```text
add地址未知
```

---

汇编器生成：

```text
需要修正的位置
```

记录：

```
Relocation Table
```

---

链接阶段：
```
add最终地址确定
↓
修改BL指令
↓
正确跳转
```

---

# 10. 目标文件（.o）的结构

一个典型 ELF 目标文件：

```
main.o

+----------------+
| ELF Header     |
+----------------+
| .text          |
| 代码段          |
+----------------+
| .data          |
| 已初始化数据    |
+----------------+
| .bss           |
| 未初始化数据    |
+----------------+
| Symbol Table   |
| 符号表          |
+----------------+
| Relocation     |
| 重定位表        |
+----------------+
| Debug Info     |
| 调试信息        |
+----------------+
```

---

# 11. .text代码段

保存：

```text
程序指令
```

例如：

```asm
MOV r0,#1

ADD r1,r0,r2
```

转换：

```
机器码
```

存放：

```
.text
```

---

# 12. .data数据段

保存：

已经初始化的全局变量。

例如：

```c
int count = 10;
```

生成：

```
.data

count:
10
```

---

# 13. .bss段

保存：

未初始化变量。

例如：

```c
int buffer[1024];
```

不会存储1024个0。

只记录：

```
需要1024字节空间
```

启动时：

由系统初始化为0。


# 18. 汇编阶段总结

汇编阶段完成：

```
汇编代码

      ↓

解析指令

      ↓

编码机器码

      ↓

生成目标文件
```

核心任务：

| 任务    | 作用     |
| ----- | ------ |
| 指令解析  | 理解汇编语法 |
| 编码    | 生成机器码  |
| 符号记录  | 记录函数变量 |
| 重定位记录 | 等待链接修正 |
| 生成.o  | 提供链接输入 |

最终结果：

```
main.s

↓

Assembler

↓

main.o
```

此时程序已经变成：

```text
CPU可以执行的机器指令
```

但还不能直接运行。

下一阶段：

```
链接(Linking)

负责：

.o文件合并
符号解析
地址分配
生成最终ELF
```
