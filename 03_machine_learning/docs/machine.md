# 机器学习是什么

机器学习是让计算机从数据中挖掘规律，而不是由程序员手工编写所有规则

## 传统编程

程序员心里有一套规则，按照这个规则来编程，就能解决对应的问题，如下所示，判断人是否成年，计算机只是负责执行

```python
if age >= 18:
    print("成年人")
else:
    print("未成年人")
```

**当规则很复杂的时候**
程序员就很难制定规则了

```text
给你关于一个房子的
面积，楼层，地段，学区，装修，朝向，交通等
你能够预测出准确的房价么
```

程序员不是中介，可能要去了解很多售楼相关的信息才能知晓，这样短时期就难以实现了

## 规则是什么

我们可以将规则理解成一套映射关系，类似于函数那样，根据输入就可以得到输出

## 机器学习的本质就是寻找规则

站在程序员的视角解决问题
**规则 + 数据 -> 结果**

机器学习的视角解决问题

（1）首先要知道映射关系（即规则）
**数据 + 结果 -> 规则**

（2）根据规则来预测结果（即推理）
**规则 + 输入 -> 规则**

# 机器如何寻找规则

引入问题：根据一个人的身高来预测其体重

我们知道，在一般情况下，同一类别的样本里面【相同性别，年龄段，健康状况良好】，身高和体重是呈现正相关的，即

```c
y = kx + b
y 是体重，x 是身高，b 是偏置
```

如果计算机知道了 k 和 b，那么它也就能够解决这个问题

**注意： 寻找的规律是适用于一般情况的规律**
不排除有些样本可能很高，但是也很瘦

## 模型

```c
y = kx + b
```

怎么理解模型？
可以理解为不同的问题它们的映射关系是不同的，上面那个简单的问题，就可以通过一元一次函数的模型来解决。比如股票的预测，这种复杂的问题，还能通过上面的模型来解决么，显然不行。

## 如何选择模型

所以模型可以简单理解为一个函数，不同的函数对应的是不同的模型【一次函数，二次函数，指数函数等】，用错了模型可能解决不了问题【比如上面的问题，你使用二次函数的模型可能就解决不了问题】

**选择合适的模型很重要，问题简单，模型复杂会导致过拟合，问题复杂，模型简单会导致欠拟合**

## 训练

知道模型之后，如何让模型适用于我们当前的问题场景呢。需要**训练**【可以理解为确定模型中的参数】，当同一个模型去解决不同场景的问题时候可能模型中的细节是不一样的，这些需要后期训练决定。【比如根据身高预测体重，根据商品销售量预测利润等，这些都可以通过一次函数的模型来解决，但是其中的细节可能不同，即模型的**参数**不同】

## 如何训练
根据真实值与预测值的误差，更新参数的变化。
对于参数k，当我们发现随着k的变大，误差越来越大时，我们应该把k变小，反之亦然

# 机器学习的三大任务

即程序员使用机器学习来完成的主要任务

（1）回归【预测数值】

```text
房价
气温
销量
股票波动
```

（2）分类【预测类别】

```text
垃圾邮件？
是否患病？
猫还是狗？
```

（3）聚类【自动分组】

```text
垃圾邮件？
是否患病？
猫还是狗？
```

# 机器学习的分类

机器学习

├── 监督学习

├── 无监督学习

└── 强化学习

## 监督学习

训练数据包含标准答案。

例如：

面积 → 房价

图片 → 标签

语音 → 文本

**回归问题：输出连续值**

常见模型
Linear Regression
Ridge
Lasso
Neural Network

**分类问题：输出类别**

常见模型
Logistic Regression
SVM
Decision Tree
Random Forest
Neural Network

## 无监督学习

没有标准答案，只有输入数据，机器自己寻找规律。

**聚类问题：自动分组**

使用场景：用户画像

常见算法
K-Means
DBSCAN
GMM

**降维：高维数据压缩**

常见算法

PCA
t-SNE
UMAP

**异常检测：发现异常样本**

比如
网络攻击
故障检测
金融欺诈

# 常见机器学习模型

| 模型 | 英文名称 | 诞生时间 | 提出背景 | 学习类型 | 主要任务 | 典型应用 |
|----|----|----|----|----|----|----|
| 线性回归 | Linear Regression | 1805年（最小二乘法提出） | 为了解决变量之间线性关系建模问题，通过寻找最佳拟合直线描述输入和输出之间的关系 | 监督学习 | 回归预测 | 房价预测、销量预测、趋势分析、经济预测 |
| 逻辑回归 | Logistic Regression | 20世纪50年代 | 在线性回归基础上发展，用于解决分类问题，通过概率形式预测类别 | 监督学习 | 分类任务（二分类、多分类） | 垃圾邮件识别、疾病预测、信用风险评估 |
| 感知机 | Perceptron | 1957年 | Rosenblatt提出，用于模拟生物神经元，实现机器自动分类，是人工神经网络的起点 | 监督学习 | 二分类 | 简单图像识别、模式分类、神经网络基础研究 |
| 多层感知机 | Multi Layer Perceptron（MLP） | 1980年代 | 为解决单层感知机无法处理非线性问题，引入隐藏层和反向传播算法 | 深度学习 | 分类、回归、特征学习 | 图像分类、语音识别、信号处理 |
| 支持向量机 | Support Vector Machine（SVM） | 1995年 | Vapnik提出，基于统计学习理论，通过寻找最大间隔超平面提升模型泛化能力 | 监督学习 | 分类、回归 | 文本分类、人脸识别、生物信息分析 |
| 决策树 | Decision Tree | 1960~1980年代 | 模拟人类决策过程，通过不断划分数据形成树形结构，实现分类和预测 | 监督学习 | 分类、回归 | 医疗诊断、用户分析、金融审批 |
| 随机森林 | Random Forest | 2001年 | Breiman提出，通过构建大量随机决策树并投票，降低单棵树过拟合问题 | 集成学习 | 分类、回归 | 风险预测、特征分析、工业预测 |
| XGBoost | Extreme Gradient Boosting | 2014年 | 在GBDT基础上优化，通过梯度提升和工程优化，提高模型训练速度和预测精度 | 集成学习 | 分类、回归 | Kaggle竞赛、推荐系统、金融风控 |
