# 随机森林（Random Forest）详解

# 1. 什么是随机森林

随机森林（Random Forest）是一种基于决策树（Decision Tree）的集成学习（Ensemble Learning）算法。

其核心思想：

```text
一棵树容易犯错
↓
训练很多棵树
↓
让它们投票
↓
得到最终结果
```

因此：

```text
随机森林 = 多棵决策树 + 随机采样 + 投票机制
```

---

# 2. 随机森林产生的背景

## 2.1 决策树的问题

决策树具有很多优点：

* 易理解
* 可解释
* 能处理非线性问题
* 不需要特征归一化

例如：

```text
年龄 > 18 ?
    ├─ 是 → 成年人
    └─ 否 → 未成年人
```

但是决策树有一个严重缺点：

```text
容易过拟合（Overfitting）
```

例如：

训练集：

```text
身高 体重 性别

170 70 男
180 80 男
160 50 女
```

如果树不断分裂：

```text
身高 > 170.2 ?
体重 > 69.8 ?
```

最终会把训练数据记住。

结果：

```text
训练准确率100%
测试准确率很低
```

---

## 2.2 集成学习思想

研究人员发现：

```text
多个弱模型
↓
组合起来
↓
可能变成强模型
```

例如：

```text
树1 正确率 70%
树2 正确率 72%
树3 正确率 68%
...
```

投票后：

```text
正确率 90%+
```

这就是集成学习。

---

## 2.3 Bagging思想

1996年，

Leo Breiman 提出：

```text
Bootstrap Aggregating
简称 Bagging
```

思路：

```text
从训练集随机抽样
↓
训练多棵树
↓
结果投票
```

随后进一步发展为：

```text
Random Forest
随机森林
```

# 3. 随机森林的核心思想

随机森林包含两个随机：

```text
随机样本
随机特征
```

因此称为：

```text
Random Forest
```

---

# 4. 整体结构

假设：

```text
训练集
10000个样本
20个特征
```

训练：

```text
数据集
 ↓
随机抽样
 ↓
树1

随机抽样
 ↓
树2

随机抽样
 ↓
树3

...

随机抽样
 ↓
树100
```

预测：

```text
树1 → 猫
树2 → 狗
树3 → 猫
树4 → 猫
...
```

最终：

```text
猫票数最多
```

输出：

```text
猫
```

---

# 5. Bootstrap抽样

这是随机森林的第一层随机性。

---

## 原始数据

```text
样本数 N=1000
```

---

## 有放回抽样

随机抽：

```text
1
5
10
10
23
45
...
```

注意：

```text
允许重复
```

例如：

```text
样本10
```

可能被抽到多次。

---

## 结果

每棵树看到的数据不同：

```text
Tree1
↓

数据集A

Tree2
↓

数据集B

Tree3
↓

数据集C
```

因此：

```text
树之间存在差异
```

---

# 6. 特征随机选择

第二层随机性。

---

例如：

```text
20个特征
```

```text
年龄
身高
体重
收入
学历
...
```

普通决策树：

```text
每次分裂
查看全部20个特征
```

随机森林：

```text
每次分裂
随机选5个特征
```

例如：

```text
年龄
体重
学历
收入
房产
```

然后：

```text
仅在这5个特征中寻找最佳分裂
```

---

# 7. 为什么要随机特征

如果不随机：

```text
所有树都会选择
年龄
```

导致：

```text
树长得很像
```

效果：

```text
投票意义不大
```

随机特征后：

```text
树之间差异变大
```

提升泛化能力。

# 10. 数学原理

设：

```math
T_1(x)
```

表示第一棵树。

```math
T_2(x)
```

表示第二棵树。

...

```math
T_n(x)
```

表示第n棵树。

---

分类：

```math
\hat y
=
\text{MajorityVote}
(
T_1,
T_2,
...
T_n
)
```

即：

```text
多数投票
```

---

回归：

```math
\hat y
=
\frac1N
\sum_{i=1}^{N}
T_i(x)
```

即：

```text
求平均值
```

# 11. 为什么随机森林效果好

机器学习误差：

```text
Bias
偏差

Variance
方差
```

---

决策树：

```text
偏差低

方差高
```

即：

```text
容易过拟合
```

---

随机森林：

通过平均多个模型：

```math
Variance
↓
```

因此：

```text
泛化能力提高
```

---

# 12. OOB（袋外误差）

Bootstrap抽样后：

```text
约36.8%的样本
不会被抽中
```

称为：

```text
Out Of Bag
OOB
```

---

用途：

```text
不需要验证集
即可评估模型性能
```

---

# 13. 特征重要性

随机森林能够评估：

```text
哪些特征最重要
```

例如：

```text
年龄      40%
收入      30%
学历      15%
体重      10%
身高       5%
```

---

这在工程中非常有价值。

# 14. 关键参数

## n_estimators

树数量

```python
RandomForestClassifier(
    n_estimators=100
)
```

通常：

```text
100~500
```

---

## max_depth

树最大深度

```python
max_depth=10
```

---

## min_samples_split

最小分裂样本数

```python
min_samples_split=5
```

---

## max_features

每次随机选多少特征

```python
max_features="sqrt"
```

分类问题最常用。

---

# 15. Python示例

```python
from sklearn.ensemble import RandomForestClassifier

model = RandomForestClassifier(
    n_estimators=100,
    max_depth=10,
    random_state=0
)

model.fit(X_train, y_train)

y_pred = model.predict(X_test)
```

# 16. 优点

## 精度高

相比单棵树：

```text
更稳定
```

---

## 不容易过拟合

多棵树平均：

```text
降低方差
```

---

## 能处理非线性

例如：

```text
图像
金融
医疗
```

---

## 可并行训练

每棵树互不影响。

---

## 可解释

支持：

```text
Feature Importance
```

---

# 17. 缺点

## 模型较大

例如：

```text
1000棵树
```

内存消耗较高。

---

## 推理速度慢

相比：

```text
逻辑回归
```

更慢。

---

## 可解释性弱于单棵树

单树：

```text
一条路径即可解释
```

随机森林：

```text
上百棵树共同决策
```

难以完全解释。

# 20. 总结

随机森林的本质：

```text
多棵决策树
+
Bootstrap抽样
+
随机特征选择
+
投票机制
```

核心目标：

```text
降低过拟合
提高泛化能力
```

一句话理解：

```text
决策树像一个专家。

随机森林像100个专家一起开会投票。

单个专家可能出错，
但多数专家同时出错的概率很低。
```
